01. Marxent Converter

Select Folder and Batch Convert

  • Creates a dialog box with two buttons: "Get maxfile folder" and "Batch Convert".
  • "Get maxfile folder" lets the user select a folder containing .max files.
  • "Batch Convert" initiates the batch conversion process.

Batch conversion process:

  • Loads the max file.
  • Cleans up the scene by un-hiding objects, removing helpers, and converting objects to editable poly format.
  • Adjusts object pivots and attaches objects together.
  • Sets up material IDs and assigns materials.
  • Adjusts UVW mapping for texture coordinates. First UV channel it set for taking maximum 0-1 space. And for 2nd UV channel it sets the texel density 2048 as per marxent standard.
  • Exports the model to the FBX format, in separate folder.
  • Saves the modified scene file, in separate folder.

User feedback

  • After processing all files, the script displays a message confirming completion of the conversion process.

02. HNI Toolset

Summary

This script offers a comprehensive toolset for model validation and correction, focusing on geometry integrity, naming conventions, unit settings, and scene organization within 3ds Max. Each tool provides feedback through message boxes and logs to ensure clarity on the status and results of the checks performed. And make log in text file for easy checking.

Detailed Functionality

  • N-gon Check: Checks for faces with more than 4 edges (N-gons) in the scene.
  • Dummy Check: Verifies the presence and position of dummy objects.
  • Layer Check: Ensures that object names match their layer names.
  • Pivot Check: Checks if object pivots are centered at the grid.
  • Unit Check: Displays the current system units.
  • Polycount: Computes and displays the total polycount of the scene.
  • STP Check: Checks for the presence of an "STP" layer.
  • Backface Culling On: Enables backface culling for all objects and layers.
  • Task Height Check: Creates reference boxes for task seat and arm heights to verify their correctness.
  • Stool Height Check: Creates reference boxes for stool seat and arm heights to verify their correctness.
  • Non-Smooth Faces Check: Checks for faces without smoothing groups.
  • Flip normal Check: Lets you easily check for flip normals by turing on Flip normals in xview.
  • Check isolated vertices: Check and remove any isolated vertices.
  • Render UV: Renders the Current UV in 0-1 space and save render in same max file folder.
  • Gap check toggle: Lets you easily check for gaps by turing on viewport background to green. Click again for turning off green background.

03. Preview Render

Purpose:

This script is used to automate the process of rendering images from 3D models. To helps save time and ensures consistency by auto setting lighting and cemaera. Then followed by batch rendering and saving images.

Features:

  • Single Render: Renders an image of the current 3D model as seen from the active viewpoint.
  • Select image save folder: Allows you to select a folder for saving rendered images.
  • 360 Renders: Renders the current 3D model from 12 angles, creating a series of images that show the model from all sides. And saves images in selected folder.
  • Get files: Allows you to select a folder containing max files for batch rendering.
  • Batch render: This will render all max files in selected filder and saves its images in selected folder.

Benefits:

  • Time-Saving: Automates repetitive tasks, freeing up time for more creative work.
  • Consistency: Ensures all your renders have the same lighting and camera setup, making your images look professional and uniform.

04. UV Tool

Main Features:

  • Get Scale: Calculates the current texel density (how much texture detail there is per unit of the model).
  • Set Scale: Allows you to set a desired texel density for selected models, ensuring consistent texture quality.
  • Set 2048: Quickly sets the texel density to a value suitable for 2048x2048 textures.
  • Align Elements: Aligns selected UV elements to ensure they are properly positioned.
  • Flat UV: Flattens the UV map in w direction. Making it easier for correct texel density calculations.

05. Outline Renderer

Model Preparation Helper:

This part of the script is for preparing models for rendering.

  • Import or merge the model.
  • For models other than chairs, click on "Add Dummy to Tables" to align your model to the origin.
  • For chair models:
    • Set the caster rotation pivot properly.
    • If there are multiple objects in one caster, link all other objects in the caster to the rotation object in the caster.
    • Select all caster heads, then click "Add Dummy to Caster".
    • Select the cylinder in the chair and click "Add Dummy to Cylinder". If there are no casters, you can skip "Add Dummy to Caster" and directly click "Add Dummy to Cylinder".
  • Rename your model parts to match their material names. Enter the name in the text field and click the "Rename It" button.
  • Select the type of product in the camera add section. If the model type is not listed, select the custom option.
  • Randomize the wirecolor of objects (material-wise) using the "Random Wirecolor" button.
  • To add UVW map to laminates, select laminates and click "Add Laminates UVW" button.
  • Once the material names are assigned to scene objects, click "Assign Material" to auto-add materials to scene objects.

Outline Render Helper:

Use models prepared with the model preparation helper.

  • Select the angle needed to render using the check button, then click "Create Keys".
  • Click "Assign Render Setup" to add the render setting and background to the scene.
  • To render preview images, click "R1 Clay".
  • To render final images in 6k resolution, check the "6K" button and click "R2 Final".
  • To render a single product, select "Local" or "Network" as your render destination, then select the image save folder and click "Render".
  • To batch merge multiple files prepared using the model preparation helper, select the folder containing those files and click "Batch Merge Max Files". Then submit to render as above.

07. Batch Mid-High Poly Export

What the Script Does in Detail:

  • Prepare Models:
    • Unhides all objects and layers in the scene.
    • Adjusts object names based on their layer names.
    • Compares "dummy" objects to determine their linked mesh counts and names them accordingly (High Poly or Mid Poly).
  • Export Models:
    • Sets parameters for FBX export.
    • Processes each 3ds Max file:
      • Separates the model into High Poly and Mid Poly versions.
      • Adjusts the position and pivot points of the models.
      • Attaches objects in the same layer.
      • Saves and exports the High Poly and Mid Poly versions as .max and .fbx files to separate folder.
  • Cleanup:
    • Removes empty layers and unused objects from the scene.
    • Displays a message once the conversion process is complete.

08. Batch Import STP/FBX/DWG Script

Features:

  • Select Folder:
    • Allows you to choose a folder containing FBX/STP/DWG files that you want to import.
  • Batch Merge:
    • Imports all FBX files from the selected folder.
    • Unhides all objects and layers in the scene.
    • Adjusts object names based on their layer names.
    • Gives you option to groups and offsets the imported objects to avoid overlap.
    • Sets pivots to the center and zeroes out the position for alignment.
    • Removes empty layers and organizes each imported file into a new layer with the file's name.
    • Displays a message once the merging process is complete.
  • Batch Save Sets:
    • Saves each selection set into a separate file.
    • Files are saved in the same location as the original Max file within a new folder named after the Max file.
    • Displays a message once the save process is complete.

09. Batch Convert Max Files to FBX

What the Script Does in Detail:

  • Folder Selection:
    • Opens a dialog for you to select the folder containing your .max files.
    • Stores the path of the selected folder and retrieves a list of all .max files within it.
  • Batch Conversion Process:
    • Opens each .max file one by one.
    • Cleans up the scene by removing helpers and resetting transformations.
    • Converts all objects to editable poly for compatibility.
    • Sets FBX export parameters such as enabling smoothing groups and setting the up-axis to Z.
    • Exports the prepared file as an FBX.
    • Deletes the objects from the scene to prepare for the next file.
    • Resets the 3ds Max scene after all files are processed.

10. Batch Mid Poly Export

What the Script Does in Detail:

  • Folder Selection:
    • Opens a dialog for you to select the folder containing your .max files.
    • Stores the path of the selected folder and retrieves a list of all .max files within it.
    • Creates a subfolder named "Mid" within the selected folder to store the converted files.
  • Batch Conversion Process:
    • Opens each .max file one by one.
    • Removes layers and objects named "STP" and "DWG".
    • Prepares the objects for export by converting them to editable poly, resetting transformations, and assigning default properties.
    • Sets the layer names as object names for better organization.
    • Attaches objects within the same layer to minimize the number of nodes.
    • Saves the prepared file as a new .max file in the "Mid" subfolder.
    • Configures FBX export settings, such as enabling smoothing groups and setting the up-axis to Z.
    • Exports the prepared file as an FBX with a standardized naming convention in the "Mid" subfolder.
    • Deletes all objects from the scene to prepare for the next file.
    • Resets the 3ds Max scene after all files are processed.

11. Batch File Separate

What the Script Does in Detail:

  • Folder Selection:
    • Opens a dialog for you to select the folder containing your .max files.
    • Stores the path of the selected folder and retrieves a list of all .max files within it.
    • Creates subfolders named "Low", "Mid", and "High" within the selected folder to store the separated files.
  • Batch Separation Process:
    • Opens each .max file one by one.
    • Removes layers and objects named "STP".
    • Selects and prepares objects for separation by converting them to editable poly, resetting transformations, and setting default properties.
    • Assigns the layer names to the object names for better organization.
    • Attaches objects within the same layer to minimize the number of nodes.
    • Compares the number of polygons in the children of three dummy objects to determine which dummy represents high-poly, mid-poly, and low-poly versions.
    • Sets the names of the dummy objects accordingly (HP, MP, LP).
    • Separates and prepares each version for export:
      • High-poly version:
        • Moves to origin, isolates, and attaches objects, sets pivots, and exports to FBX and .max files in the "High" subfolder.
      • Mid-poly version:
        • Moves to origin, isolates, and attaches objects, sets pivots, and exports to FBX and .max files in the "Mid" subfolder.
      • Low-poly version:
        • Moves to origin, isolates, and attaches objects, sets pivots, and exports to FBX and .max files in the "Low" subfolder.
    • Deletes all objects from the scene to prepare for the next file.
    • Resets the 3ds Max scene after all files are processed.

What the Script Does in Detail:

17. Batch PSD to HON (Photoshop script)

How It Works:

  • Folder Selection:
    • The user selects a folder containing the PSD files to process.
    • If a folder is chosen, a new subfolder named "PSD converted" is created within it to store the processed files.
  • File Processing:
    • The script goes through each PSD file in the selected folder.
    • For each PSD file:
      • Open the File: The PSD file is opened in Photoshop.
      • Process Layers and Layer Sets:
        • Alpha Layer Set: Removes all layers except those containing "multiMatte" in their name.
        • Selection Layer Set: Moves a layer named "multiMatte" to the "wirecolor" layer set (if it exists).
        • Removal of Layer Sets: Removes any layer sets named "selection" or "swatch".
        • Merge Layers in Specific Sets: Merges layers within certain sets into single layers.
        • Turn Off Visibility: Makes certain layers invisible.
      • Save and Close the File:
        • The processed file is saved to the "PSD converted" subfolder with the same name as the original file.
        • The file is then closed without saving any further changes.

01. Smart Renamer

Key Features:

  • Custom Naming:
    • You can enter a custom name for selected objects.
  • Automatic Layer and Color Assignment:
    • When you rename objects, the tool can automatically assign them to new layers and give them random wireframe colors for easy identification.
  • Presets for Common Elements:
    • There are buttons for quickly renaming objects to common architectural elements like walls, floors, windows, etc. Each button:
      • Assigns a specific name prefix (e.g., "Wall_stucco_").
      • Changes the wireframe color to a predefined value.
      • Optionally adds a UVW map modifier and assigns a specific material.
  • Material and UVW Map Management:
    • The tool can automatically apply materials from a predefined library.
    • It can add UVW map modifiers to objects to ensure textures are mapped correctly.
  • Utility Functions:
    • You can remove materials and UVW maps from selected objects with one click.
    • There's a function to delete empty layers in your scene to keep things tidy.

05. Vikrell Keyframer

Here's a summary of what it does in simpler terms:

  1. User Interface: The script creates a window (UI) where you can see various checkboxes and buttons organized neatly.
  2. Object Visibility: It helps control the visibility of different objects in a 3D scene across multiple frames of animation.
  3. Checkbox Controls: Each checkbox corresponds to a frame of animation (e.g., Frame 1, Frame 2, etc.), and each column of checkboxes corresponds to a specific object or group of objects in the scene (e.g., Tub, Shower Base, Grab Bars, etc.).
  4. Adding Keys: By checking or unchecking the checkboxes and pressing the "Add keys" button, you can create animation keys that control whether each object is visible or not at different frames.
  5. Other Features:
    • Buttons like "Shower model" and "Tub model" toggle the visibility of specific objects instantly.
    • "Exclude" button excludes certain types of objects from affecting lighting in the scene.
    • "Visible" and "Invisible" buttons quickly add visibility keys for currently selected objects.
    • "Delete All keys" button removes all visibility keys from the scene.
    • Checkboxes to hide or show lights in the scene.

06. Vikrell Renamer

It's a tool for managing and organizing different parts of a digital design project, specifically related to Vikrell products.

  • Interface: Opens a small window with buttons for components like walls, tubs, shower bases, etc.
  • Renaming: Clicking a button renames selected objects in the scene to start with a specific prefix followed by a unique identifier (e.g., "Tub_").
  • Material Assignment: Assigns materials to objects; for example, selecting "Tub" applies a material called "Studio 10b-White Tub" to tub objects.
  • Additional Features: Some buttons perform extra actions, like "Backer board," which not only renames and assigns materials but also adds a mapping modifier to selected objects.

08. Viewport Material Change

Description:

This script creates a button labeled "Color change". When clicked, it toggles the display color mode between showing the color of objects in wireframe and showing the of object materials in viewport. Useful in product modelling.

09. Bridge Poly Script

Description:

This script enables a hotkey for the Bridge Poly operation in 3ds Max. Previously, this functionality was unavailable, but now it's accessible by default.

10. Align Pivot to Bottom

Description:

This script aligns the pivot of selected objects to the minimum z distance of objects. It also position pivot at the center of the object.

11. Viewport Auto Switch to 2D and 3D

Description:

This script adjusts snapping behavior in the design software based on the viewing mode. It automatically switches between full 3D snapping and 2.5D snapping depending on the view, ensuring effective snapping functionality tailored to the design context. Very effective tool in architectural modelling.

12. Vray Physical Cam Restore

Description:

This script restores the old physical camera functionality that was present in Vray before 2018. It reverts to the previous version of the Vray physical camera, allowing users to access and utilize the features and settings that were available prior to the 2018 update.

13. Random Material Apply

Description:

This script enables the random application of materials from the material editor window to selected objects in the scene. It automates the process of assigning materials to objects by selecting them randomly from the available materials in the material editor.

14. Animation Keys Remover

Description:

This script is designed to remove all animation keys associated with selected objects in the scene. It provides a convenient way to clear animation data from objects, allowing users to start fresh or make adjustments without the constraints of existing keyframes.
